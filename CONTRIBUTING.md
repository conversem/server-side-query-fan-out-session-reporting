# Contributing to LLM Bot Traffic Pipeline

Thank you for your interest in contributing! This is an open source project and we
welcome community contributions.

## Quick Start

See [docs/development/contributing.md](docs/development/contributing.md) for the full
contributing guide, including:

- Development environment setup
- Code standards (black, isort, type hints)
- How to submit pull requests
- Testing requirements
- How to add new ingestion providers or storage backends
- Security policy

## Summary

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/my-feature`)
3. Make your changes with tests
4. Format: `black src/ scripts/ tests/ && isort src/ scripts/ tests/`
5. Run tests: `pytest`
6. Open a pull request against `main`

## Reporting Issues

Use [GitHub Issues](https://github.com/conversem/server-side-query-fan-out-session-reporting/issues).
Search for existing issues before creating a new one.

## Security Vulnerabilities

Report security issues privately via
[GitHub Security Advisories](https://github.com/conversem/server-side-query-fan-out-session-reporting/security)
— do **not** open a public issue for security vulnerabilities.
